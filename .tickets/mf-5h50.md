# Whitespace skipping is too broad — accepts non-spec whitespace chars

Parser uses Rust's ch.is_whitespace() via is_ignorable_char for skip_whitespace, which accepts Unicode whitespace beyond the ABNF ws production (SP / HTAB / CR / LF / %x3000). Characters like NBSP (U+00A0), VT, FF, NEL, U+2000-U+200A, U+2028-U+2029 are silently accepted as whitespace. Should use is_mf2_whitespace from charset.rs (which is correct) instead of ch.is_whitespace().
