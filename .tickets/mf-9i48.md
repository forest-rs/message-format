# Remove non-spec -> select expression syntax

The parser supports a custom -> inline select syntax: { $x :plural -> [one]one *[other]other }. This has no equivalent in the TR35 MF2 ABNF and diverges from the spec. With .match fully implemented, this extension is unnecessary. Remove parse_select_after_operand and all -> checks in parse_expression_interior. Clean up SelectExpressionNode, SelectArmNode, and ExpressionKindNode::Select if they become unused.
