# Bidi-only sequences should not satisfy mandatory whitespace (s production)

ABNF s production requires at least one actual ws character; bidi controls alone do not satisfy it. Currently skip_whitespace returns true when only bidi controls were consumed (via is_ignorable_char), so a bidi-only sequence between tokens incorrectly passes mandatory-whitespace gates. The s production is: s = *bidi ws o, requiring at least one ws.
